# Project Rules for Nicolas Portfolio (React)

This project is a fully customized frontend developer portfolio built with React, representing the professional identity of Nicolas.

## Goals
- Showcase frontend skills with clean, scalable and modern code.
- Allow easy expansion with new technologies, tools, or styles.
- Serve as a reference for best practices and code quality.

## Rules

- Framework: React with Vite.
- Language: TypeScript.
- Styling: TailwindCSS preferred, but other styles may be integrated per section (e.g., styled-components for animations).
- Folder Structure: Use domain-based or atomic folder structure.
- State Management: Prefer local state and context API. Redux or Zustand only if scaling requires it.
- Components: Must be reusable and follow Single Responsibility Principle (SRP).
- Performance: Use lazy loading for non-critical components. Optimize images and assets.
- SEO: Use React Helmet for meta tags.
- Animations: Use Framer Motion when necessary, keep animations lightweight.
- Deployment: Deploy with Vercel or Netlify.
- Version Control: Git with proper commit messages (Conventional Commits recommended).
- CI/CD: Prepare for GitHub Actions integration.
- Documentation: Maintain a README and brief documentation per feature/component if necessary.
